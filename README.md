# Прогноз негативных отзывов в мобильном приложении

## Описание проекта

**Цель:** Разработать модель машинного обучения, которая по поведенческим метрикам пользователя использования мобильного приложения предсказывает, оставит ли он негативный отзыв (оценка 1–2 звезды).

**Бизнес-задача:**  
Снизить долю негативных отзывов, автоматически выявляя пользователей с высоким риском и попытаться удержать (например, отправляя им пуш-уведомления с предложением помощи или промокодом).

## Данные

Данные синтезированы на основе реалистичных поведенческих паттернов. Генерация выполнена с помощью статистических распределений (отрицательное биномиальное, экспоненциальное, Пуассона, равномерное) с заложенными зависимостями между признаками и целевой переменной.

| Признак | Описание |
|---------|----------|
| `total_sessions` | Общее количество сессий за первые 3 дня |
| `total_time_min` | Общее время в приложении (минуты) |
| `count_errors` | Количество ошибок/сбоев |
| `completed_onboarding` | Флаг завершения онбординга (0/1) |
| `filled_profile` | Флаг заполнения профиля (0/1) |
| `used_search` | Флаг использования поиска (0/1) |
| `num_screens_visited` | Количество уникальных экранов |
| `bad_review` | **Целевая переменная** – негативный отзыв (1) или нет (0) |

**Дисбаланс классов:** ~7% пользователей оставляют негативные отзывы, что соответствует реальным данным мобильных приложений.

## Этапы работы

### Исследовательский анализ данных (EDA)
- Проверка пропусков и типов данных.
- Анализ распределения целевой переменной.
- Сравнение средних значений признаков для классов.
- Визуализация гистограмм и boxplot.
- Построение корреляционной матрицы.

### Создание новых признаков (Feature Engineering)
- `error_rate` – ошибок на сессию.
- `avg_session_time` – средняя длительность сессии.
- `power_user` – флаг для активных пользователей.
- `low_engagement` – флаг низкой активности.
- `pain_index` – составной индекс «боли» (ошибки + штраф за незавершённый онбординг).
- `time_per_screen` – время на экран.
- `log_total_time` – логарифм общего времени.

### Обучение моделей
Сравнивались три подхода:
- **Logistic Regression** без балансировки (baseline).
- **Logistic Regression** с `class_weight='balanced'`.
- **Random Forest** с `class_weight='balanced'` и настройкой гиперпараметров.

### Оценка качества
- Матрицы ошибок.
- ROC-кривые и AUC.
- Precision-Recall кривые.
- Feature Importance для Random Forest.
- Бизнес-метрика ROI (прибыль от предотвращённых негативных отзывов).

##  Результаты

| Модель | Recall (класс 1) | Precision (класс 1) | F1 | Прибыль (у.е.) |
|--------|------------------|---------------------|----|----------------|
| LR без балансировки | 0.03 | 0.67 | 0.06 | ~9.9 |
| LR с балансировкой | 0.71 | 0.18 | 0.29 | 223.0 |
| **Random Forest (лучший)** | **0.80** | **0.18** | **0.30** | **514.9** |

**Лучшая модель:** Random Forest с `class_weight='balanced'`, `max_depth=3`.

**Ключевые признаки:**
1. `count_errors` – количество ошибок (главный драйвер негатива).
2. `pain_index` – индекс «боли».
3. `avg_session_time` – среднее время сессии.
4. `total_sessions` – общее количество сессий.
